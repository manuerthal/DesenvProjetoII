Controller - Definição de rotas, destacando o seu tipo e funções associadas a mesma
Repository - Arquivo onde as funções de contato com banco de dados são "criadas", neste caso usamos o JPA repository, modelo
com funções ja prontas de select, save e delete
DTO- Classe modelo para receber os dados do frontend, utilizamos verificações de validação para ter certeza que os dados vem
corretos
Model- Classe modelo para criação da tabela no banco de dados e para alterações com o mesmo, ja que é necessario passar uma classe
específica nas funções do repository
Service - Meio termo entre o controller e o repository, onde são aplicados as regras, como por exemplo adicionar ao registro
a data atual
Controller - recebe infos no link definido
Service - trata as infos
Repository - Altera as infos no banco de dados
SpringBoot - framework que facilita a criação de API, como esta que é uma REST API
Dependencias - Bibliotecas importadas que são necessarias para utilização do app desenvolvido
